## 3. PNG 익스포트 → promo-automation 규칙 이식

**변경 파일**
- `lib/export-naming.ts` (신규) — `mmdd()`, `promotionSlug()` (promo `SAFE_FILENAME_RE` 그대로), `landingBaseName()`, `blockImgName()`
- `app/render/[id]/page.tsx` — 각 블록을 `data-block-slot` + `data-block-id` + `data-block-index` 로 래핑
- `app/api/pages/[id]/export/route.ts` — Playwright 로 메인 1장 + 블록별 스크린샷 → JSZip 로 폴더 구조 zip. `?format=png` 로 단일 PNG 폴백
- `app/admin/pages/[id]/Editor.tsx` — 버튼 라벨 "PNG 익스포트" → "ZIP 익스포트"

**Zip 구조 (promo-automation 그대로)**
```
{MMDD}_landing_{promotion}_{width}/
├─ {MMDD}_landing_{promotion}_{width}.png    ← 메인 (한 장 전체)
├─ img_01.png                                 ← 블록 1
├─ img_02.png                                 ← 블록 2
└─ img_03.png                                 ← ...
```

**미이식** (Landing CMS 컨텍스트상 불필요)
- Way/Jira 업로드 흐름 (Figma 플러그인 전용)
- `_수정N` 자동 채번 (Landing CMS 는 페이지 자체가 원본 관리)
- `_skip` 자식 제외 (블록 삭제로 대체)
- 3000px 자동 N등분 (블록이 이미 명시적 분할)
