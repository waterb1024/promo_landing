**Zip 구조 (promo-automation 그대로)**
```
{MMDD}_landing_{promotion}_{width}/
├─ {MMDD}_landing_{promotion}_{width}.png    ← 메인 (한 장 전체)
├─ img_01.png                                 ← 블록 1
├─ img_02.png                                 ← 블록 2
└─ img_03.png                                 ← ...
```

**미이식** (Landing CMS 컨텍스트상 불필요)

- `_수정N` 자동 채번 (Landing CMS 는 페이지 자체가 원본 관리)
- `_skip` 자식 제외 (블록 삭제로 대체)
- 3000px 자동 N등분 (블록이 이미 명시적 분할)

## 7. 라이브러리 이미지 → 랜딩 적용 시 흰 배경 자동 투명화

**변경 파일**
- `lib/image-transparency.ts` (신규) — `_remove_white_background()` TS 이식 (8-seed floodfill · thresh=32 · soften=40 anti-alias fringe)
- `app/api/assets/{id}/file/route.ts` — `?transparent=1` 쿼리 파라미터 지원. 디스크 캐시 (`{id}.transparent.png`) 로 miss→hit
- `app/admin/pages/[id]/BlockFields.tsx` — AssetPicker onSelect 에서 PNG 자산일 때 자동으로 `?transparent=1` 붙임
- `lib/assets.ts` — `deleteAsset` 이 투명 캐시도 함께 정리, `assetUrl()` 헬퍼에 `{transparent: true}` 옵션
- `package.json` — `pngjs` + `@types/pngjs`

**검증**
- 갤러리 임포트 이미지 1024×1024 · 원본 투명 0px → 변환 619,657 px 투명
- 캐시 miss 첫 요청 166ms → 캐시 hit 재요청 28ms (약 6× 속도 개선)
- 오브젝트 안쪽 흰색 픽셀은 배경과 연결 안 돼서 유지됨

## 11. Hero 이미지가 프리뷰에 안 뜨는 버그 수정 (stale closure)

**원인**
- ImageField 가 `onChange(url)` 먼저 부르고 몇백 ms 후 `onPalette(palette)` 부름
- HeroFields 의 `patch` 가 렌더 시점의 `block` 을 클로저로 캡처
- 두 번째 patch 가 `{...OLD_block, bg, labelBg}` 로 첫 번째 patch 의 `imageUrl` 을 지움

**해결**
- ImageField API 를 `onChange`(URL 변경) + `onApply`(자산 선택 시 원자적 payload) 로 분리
- HeroFields 는 onApply 에서 `patch({imageUrl, bg, labelBg})` 한 번의 호출로 처리
